Para criar um sistema de combos no seu jogo 2D em Unity, você vai precisar gerenciar tanto a lógica dos ataques em sequência quanto a transição das animações. Vou te mostrar o passo a passo básico de como implementar isso usando C#:

1. Criação dos Ataques em Sequência (Combos)

Você pode fazer isso por meio de uma sequência de estados que vai controlar o tempo de entrada de cada ataque. Quando o jogador aperta o botão de ataque várias vezes, ele executa diferentes animações, de acordo com o número de ataques.

Aqui está um exemplo básico de como implementar a lógica de combo:

a) Variáveis e Estrutura Inicial

csharp
public class PlayerCombat : MonoBehaviour
{
    public Animator animator;
    public float comboResetTime = 1f;
    private float lastAttackTime;
    private int comboStep = 0;
    private bool isAttacking = false;

    // Configurar o número de ataques no combo
    private int totalComboAttacks = 3;

    void Update()
    {
        // Detecta o botão de ataque
        if (Input.GetButtonDown("Fire1"))
        {
            Attack();
        }

        // Reseta combo se passar tempo demais entre ataques
        if (Time.time - lastAttackTime > comboResetTime)
        {
            ResetCombo();
        }
    }

    void Attack()
    {
        if (isAttacking) return;

        isAttacking = true;

        comboStep++;

        // Limita o combo ao número máximo de ataques
        if (comboStep > totalComboAttacks)
        {
            comboStep = 1;
        }

        // Chama animações específicas para cada ataque
        animator.SetTrigger("Attack" + comboStep);

        lastAttackTime = Time.time;
    }

    void ResetCombo()
    {
        comboStep = 0;
        isAttacking = false;
    }

    // Função chamada no final da animação para permitir o próximo ataque
    public void OnAttackEnd()
    {
        isAttacking = false;
    }
}

2. Gerenciar Animações no Animator

No Animator, você vai criar estados de animação para cada ataque do combo. Por exemplo:

  • Attack1
  • Attack2
  • Attack3

Conecte essas animações usando transições, baseadas no trigger "Attack1", "Attack2", "Attack3". No código acima, o parâmetro é ativado cada vez que o jogador ataca, e o número do ataque é determinado pela variável comboStep.

3. Controlar o Fim de Cada Animação

No final de cada animação de ataque, você pode usar um evento de animação para chamar a função OnAttackEnd() no script, que vai permitir que o jogador inicie o próximo ataque.

  • Na janela de animação, você pode adicionar um evento ao final da animação que chama a função OnAttackEnd.

4. Configurar as Transições

No Animator:

  • Defina que cada estado de ataque (Attack1, Attack2, etc.) tenha uma transição de volta para um estado "Idle" caso o jogador não siga com o combo.
  • Defina também transições automáticas entre Attack1 -> Attack2 -> Attack3 para quando o jogador pressionar o botão rapidamente.

Resumo:

  • Use comboStep para gerenciar qual ataque será executado.
  • Cada ataque ativa uma animação específica via triggers no Animator.
  • Utilize eventos de animação para controlar o fluxo e garantir que o jogador só possa atacar quando a animação anterior termina.

public class Espatula : ArmaPau
{
    Collider2D colisorDano;
    int dano = 9;
    float velocidadeAtaque = 2f;
    bool podeAtacar = true;

    int passoCombo = 0; // Passo atual do combo
    int totalDeCombos = 3; // Total de ataques no combo
    float tempoUltimoAtaque;
    float tempoParaResetCombo = 1f; // Tempo máximo entre ataques no combo

    private void Start()
    {
        colisorDano = GetComponent<Collider2D>();
        colisorDano.enabled = false;
    }

    public override void Update()
    {
        // Reseta o combo se o tempo máximo entre os ataques for ultrapassado
        if (Time.time - tempoUltimoAtaque > tempoParaResetCombo)
        {
            ResetarCombo();
        }
    }

    public override IEnumerator ataque()
    {
        if (!podeAtacar) yield break;

        podeAtacar = false;
        passoCombo++;

        // Limita o combo ao número máximo de ataques
        if (passoCombo > totalDeCombos)
        {
            passoCombo = 1;
        }

        // Ativa a animação do ataque correspondente ao passo do combo
        // Supondo que tenha animações "Ataque1", "Ataque2", "Ataque3" configuradas no Animator
        GetComponent<Animator>().SetTrigger("Ataque" + passoCombo);

        // Ativa o colisor para causar dano
        colisorDano.enabled = true;
        yield return new WaitForSeconds(velocidadeAtaque);
        colisorDano.enabled = false;

        podeAtacar = true;
        tempoUltimoAtaque = Time.time; // Atualiza o tempo do último ataque
    }

    private void ResetarCombo()
    {
        passoCombo = 0;
    }

    private void OnTriggerEnter2D(Collider2D collision)
    {
        // Causa dano ao inimigo caso ele colida com o colisor durante o ataque
        if (collision.GetComponent<inimigo>() != null)
        {
            collision.GetComponent<inimigo>().levaDano(dano);
        }
    }
}

  2. Reseta Combo: Se o jogador demorar muito para realizar o próximo ataque, o combo será resetado pela função ResetarCombo().
  3. Collider: O colisor (colisorDano) é ativado apenas durante o ataque, garantindo que o inimigo receba dano apenas quando o ataque está em execução.

Isso evita que múltiplos ataques sejam iniciados enquanto o jogador ainda está no meio de um ataque.

  2. Modificando Variáveis:

    • As variáveis dano, velocidadeAtaque, e totalDeCombos são definidas como protected no pai. Isso significa que elas podem ser acessadas e modificadas diretamente nas classes filhas.

Resumo:

  • Use protected para as variáveis na classe pai, permitindo que elas sejam acessadas e modificadas nas classes filhas.
  • Declare métodos na classe pai como virtual e sobrescreva-os nas classes filhas usando override, permitindo que cada arma tenha comportamento específico, mas ainda reaproveitando a lógica comum do pai.

Isso pode ser feito sem depender do Update() diretamente, controlando o tempo dentro da coroutine ou usando um temporizador antes de permitir outro ataque.

Aqui está como você pode garantir que o jogador só possa atacar novamente após o cooldown definido, impedindo que novos ataques sejam iniciados até que o intervalo de recarga entre ataques seja respeitado:

Solução: Implementando um Cooldown de Ataque

Código Atualizado com Controle de Cooldown:

csharp
public class Espatula : ArmaPau
{
    // Cooldown para o próximo ataque
    private float cooldownAtual;
    
    // Sobrescrevendo o método Start para modificar as variáveis específicas da Espátula
    protected override void Start()
    {
        base.Start();
        dano = 9; // Dano específico para a Espátula
        velocidadeAtaque = 2f; // Tempo de duração do ataque
        totalDeCombos = 2; // Espátula tem um combo de 2 ataques
        cooldownAtual = 0f; // Tempo que faltava para poder atacar de novo
    }

    // Método para chamar o ataque com controle de cooldown
    public override IEnumerator ataque()
    {
        // Se o tempo de cooldown ainda não tiver terminado, não permitir atacar
        if (!podeAtacar || cooldownAtual > 0f)
        {
            yield break;
        }

        // Atacar e iniciar cooldown
        podeAtacar = false;
        passoCombo++;

        if (passoCombo > totalDeCombos)
        {
            passoCombo = 1;
        }

        // Chama a animação correspondente ao passo do combo
        GetComponent<Animator>().SetTrigger("Ataque" + passoCombo);

        // Ativa o colisor para causar dano
        colisorDano.enabled = true;
        
        // O tempo que o colisor permanece ativado depende da velocidade do ataque
        yield return new WaitForSeconds(velocidadeAtaque);
        
        // Desativa o colisor após o ataque
        colisorDano.enabled = false;

        // Inicia o cooldown após o ataque ser realizado
        cooldownAtual = velocidadeAtaque;
        
        // Após o ataque, o jogador pode atacar novamente após o cooldown
        StartCoroutine(ResetarCooldown());

        podeAtacar = true;
    }

    // Coroutine que reseta o cooldown depois do tempo de espera
    private IEnumerator ResetarCooldown()
    {
        while (cooldownAtual > 0)
        {
            cooldownAtual -= Time.deltaTime;
            yield return null; // Espera até o próximo frame
        }
    }

    protected override void OnTriggerEnter2D(Collider2D collision)
    {
        base.OnTriggerEnter2D(collision);
        // Você pode adicionar lógica adicional ao colidir com o inimigo, como efeitos específicos
    }
}

Explicação:

  1. Cooldown de Ataque (cooldownAtual):

    • Foi adicionada uma variável cooldownAtual, que mantém o tempo restante para o jogador poder atacar novamente.
    • O ataque só é permitido se o cooldown for 0 ou menor, garantindo que o jogador respeite o intervalo entre os ataques.

  2. Controle de Cooldown:

    • Ao iniciar um ataque, o cooldown é configurado com o valor de velocidadeAtaque (o tempo que leva para o ataque ser concluído).
    • A coroutine ResetarCooldown() é chamada para diminuir o valor de cooldownAtual a cada frame, até que ele chegue a 0.

  3. Impedindo Ataques Consecutivos Sem Espera:

    • O ataque é interrompido se o cooldown ainda não tiver terminado (cooldownAtual > 0).
    • Isso garante que o jogador só possa realizar o próximo ataque depois que o tempo de cooldown for completado.

  4. Controle de Animação e Colisor:

    • A animação e o colisor continuam sendo ativados normalmente durante o ataque.
    • O yield return new WaitForSeconds(velocidadeAtaque) controla quanto tempo o colisor fica ativado para causar dano e quando o próximo ataque poderá ocorrer.

Como Funciona:

  • Quando o jogador realiza um ataque, o cooldown é ativado e o próximo ataque só será permitido após o tempo de espera.
  • Mesmo que o jogador pressione o botão de ataque repetidamente, a verificação do cooldown (cooldownAtual > 0f) impede que ele inicie outro ataque até o cooldown terminar.

Vantagens:

  • Você tem controle total sobre quanto tempo o jogador precisa esperar entre ataques.
  • As animações e a lógica de ativação do colisor continuam funcionando normalmente.
